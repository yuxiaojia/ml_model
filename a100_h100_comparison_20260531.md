# A100 vs H100 Runtime Comparison

All comparable clean rows use batch size 100 and 100 timed batches, for 10,000 CIFAR-100 images.

## Clean CUDA Event Timing

These are non-Nsight `benchmark_runtime.py` CUDA-event means.

| model | setup | A100 ms/batch | H100 ms/batch | H100 vs A100 |
|---|---|---:|---:|---:|
| resnet | PyTorch | 3.813 | 1.780 | -53.3% |
| resnet | cutlass_fresh | 6.085 | 6.290 | +3.4% |
| resnet | cutlass_test | 6.115 | 6.760 | +10.5% |
| mobilenet | PyTorch | 9.858 | 3.667 | -62.8% |
| mobilenet | cutlass_fresh | 13.535 | 13.009 | -3.9% |
| mobilenet | cutlass_test | 13.863 | 13.144 | -5.2% |
| shufflenet | PyTorch | 9.551 | 5.201 | -45.5% |
| shufflenet | cutlass_fresh | 14.837 | 7.088 | -52.2% |
| shufflenet | cutlass_test | 13.716 | 7.204 | -47.5% |

Sources:
- A100: `runtime_results_after_print_removal/`
- H100: `runtime_results_h100_20260531_redo_clean/`

## Corrected Taildup Availability

The available full-batch taildup results are not perfectly matched across GPUs:

| model | A100 taildup print-validation ms/batch | H100 taildup no-print diagnostic ms/batch | note |
|---|---:|---:|---|
| resnet | 587.306 | 19.271 | A100 includes CUDA device printf overhead |
| mobilenet | 589.404 | 28.802 | A100 includes CUDA device printf overhead |
| shufflenet | 618.417 | 16.092 | A100 includes CUDA device printf overhead |

Sources:
- A100 print-validation taildup: `runtime_results_taildup_all_batches/`
- H100 corrected taildup: `runtime_results_h100_20260531_taildup_cuda_event_1747/`

Do not use this table as a clean A100-vs-H100 taildup speedup claim, because the A100 full-batch taildup run enabled `--tail-dup-print` and is dominated by device `printf` overhead.
